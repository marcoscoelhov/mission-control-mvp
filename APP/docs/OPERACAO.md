# Operação (Monarca) — Mission Control

Este documento é para o **Marcos** operar o Reino sem depender de explicação.

## Regra-mãe
Se não dá pra provar, **não está feito**.

## Criar missão (rápido)
1) Abra **Broadcast**
2) Preencha:
   - **Contexto**
   - **Critérios de sucesso** (obrigatório)
   - **Risco** (0/1/2)
3) Envie.

### Risco (como escolher)
- **R0**: interno/seguro (mudanças locais, ajustes de UI, docs internas)
- **R1**: muda fluxo/contratos/arquitetura → exige revisão do Jarvis antes de considerar DONE
- **R2**: qualquer ação externa (redes sociais, terceiros, dinheiro) → exige OK do Marcos

## Executar uma missão
- Entre na coluna **Executando** e clique **Executar**.
- DONE só é permitido quando há **PROOF**.

## Quando algo trava
Existem dois tipos de “travado” (não confundir):

### 1) Proof pendente
Significa: rodou, mas faltou evidência verificável.
Ação:
- Ajuste o alvo (arquivo/tarefa), ou
- Anexe evidência alternativa (screenshot/log), e
- Reexecute.

### 2) Aguardando você
Significa: precisa da sua decisão/contexto.
Ação:
- Clique **Responder agora** no card.
- Responda no formato:

```
OBJETIVO: ...
ARQUIVO-ALVO: ...
SUCESSO: ...
```

O sistema move a missão de volta para **Delegadas** para reexecução.

## Comandos (substitui “chat inútil”)
- Abra **Comandos**
- Escolha o agente
- Escreva o comando
- Isso cria um card rastreável (Assigned/Delegadas)

## Onde responder (Dashboard vs WhatsApp)
- Fonte de verdade: **Dashboard**.
- WhatsApp: **alertas** (pra te avisar que existe algo pendente).
